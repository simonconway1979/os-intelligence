What's actually built — the bits that matter when you read the source.

Parallel sub-agent fan-out — for batch context enrichment; no race conditions on shared writes.
Sonnet/Opus model routing — per skill; cost-optimised capture, reasoning-optimised synthesis.
Append-only intelligence layer — with status lifecycle (raw → synthesised → validated → archived).
Schema-aware permission migration — in /os-welcome; diffs your config and asks, doesn't presume.
Local activity log, zero telemetry — opt-in /share-usage for the power-user funnel.
Versioned via git tags + migration scripts — breaking changes ship an idempotent migration, not a "good luck."

The honest answers.

What about data security?

OS-Intelligence runs on your existing Claude Code installation. Files stay on your machine. I never hold, process, or see your context.

Won't Anthropic ship this?

They'll ship better primitives over time. OS-Intelligence is the methodology — how knowledge work uses context. When Anthropic ships better foundations, OS-Intelligence gets better with them.

Is Claude Code a hard prerequisite?

Yes. If your team isn't sanctioned to use Claude Code, this isn't a fit yet.
